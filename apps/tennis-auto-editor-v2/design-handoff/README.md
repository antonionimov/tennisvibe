# Frontend Handoff

把你用其他工具做好的前端放到这个目录里。

## 推荐你给我的内容

### 方案 A，最好
直接给完整前端源码，例如：

- `index.html`
- `styles.css`
- `app.js`
- 或 React/Vue/Next/Vite 项目源码
- 图片、图标、字体等素材

### 方案 B，也可以
如果对方工具只能导出设计稿或静态文件，可以给：

- HTML + CSS + JS
- Figma 导出的页面资源
- 截图 + 标注
- 交互说明文档

## 最少要说明这几件事

1. 上传按钮对应哪个区域
2. 下载按钮对应哪个区域
3. 是否有进度状态显示
4. 是否有“处理中”文案/动画
5. 是否需要保留历史记录或项目列表
6. 移动端和桌面端是否是同一套布局

## 你可以怎么给我

### 方法 1，最省事
直接把文件放到：

`apps/tennis-auto-editor-v2/design-handoff/`

### 方法 2
如果你手里是一个压缩包，放到这个目录也可以，我来解。

### 方法 3
如果你现在只有截图或链接，也可以先发我：

- 页面截图
- 导出的 HTML
- Git 仓库地址
- 网页链接

## 我拿到后会做什么

1. 把新前端接到 `apps/tennis-auto-editor-v2`
2. 保留上传视频入口
3. 把现有测试版流程改成后台自动执行
4. 只把最终下载视频结果暴露给用户
5. 逐步补上进度状态和失败提示
